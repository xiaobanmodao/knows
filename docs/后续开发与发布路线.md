# 知识通后续开发与发布路线

## 1. 当前项目定位

知识通当前是一个微信原生小程序项目，第一阶段目标是先发布“初中数学知识库 MVP”。现有版本已经完成数学主流程：章节目录、知识点详情、题型模板、分层例题、图示展示、搜索、收藏、最近浏览和游客模式。

长期方向不是只做一个静态资料库，而是逐步发展为“初中多学科学习工具”：

1. 先发布稳定可用的数学 MVP。
2. 再补齐练习、错题、学习记录，形成学习闭环。
3. 然后扩展英语、物理，沉淀通用学科内容模型。
4. 最后加入账号同步、内容后台、智能推荐和运营专题。

## 3. 第一目标：完成发布

当前阶段的最高优先级是发布数学 MVP。发布前不要大规模扩新功能，先保证已有功能稳定、内容不出现明显占位、图片能加载、审核风险可控。

### 3.1 发布前必须完成

#### 配置确认

- 确认 `project.config.json` 中的 `appid` 是最终要发布的小程序 AppID。
- 确认微信公众平台的小程序名称、头像、简介、服务类目已经配置。
- 确认教育类目、内容展示方式和资质要求符合微信后台当前审核要求。
- 确认 `app.json` 中页面路径、tabBar、云开发开关配置正确。
- 确认 `sitemap.json` 可用，不暴露不该被索引的测试页面。

#### 云开发确认

- 确认 `utils/asset-config.js` 中的 `CLOUD_ENV_ID` 是正式云环境。
- 确认 `REMOTE_ASSET_BASE` 指向正式云存储资源根路径。
- 上传或同步需要发布的图片资源到云存储。
- 部署 `cloudfunctions/getImageTempUrls` 云函数。
- 真机验证章节图、知识点图、题目图、模板图都能正常加载。
- 如果云函数失败，要能在控制台看到明确日志，避免页面无声空图。

#### 内容确认

- 运行 `node scripts/check-math-content.js`，确保没有占位式题目、缺失模块或重复题干。
- 运行 `node scripts/check-unique-figures.js`，确保知识点和题目图存在、尺寸正确、不重复。
- 抽查每个年级至少 2 个章节，确认知识点、例题、模板文字自然可读。
- 检查页面上不要出现“测试”“占位”“待补充”“开发中但可点击”等审核敏感状态。
- 英语、物理入口保持“即将开放”，点击只提示后续开放，不进入空页面。

#### 功能确认

- 首页可以进入数学页。
- 数学页可以打开所有章节。
- 章节页可以打开知识点和题型模板。
- 知识点页可以收藏、取消收藏，并写入最近浏览。
- 收藏页可以打开收藏和最近浏览。
- 搜索页可以搜索章节名、知识点名、模型名。
- 模板页可以打开适用章节。
- 页面返回、tab 切换、下拉/滚动体验正常。

#### 真机体验确认

- 在 iPhone 和 Android 各至少测试一次。
- 检查长文本不会溢出。
- 检查图片加载速度和失败状态。
- 检查页面滚动不卡顿。
- 检查字体大小、按钮区域、卡片间距适合手机阅读。
- 检查弱网下图片加载失败时页面仍可阅读。

### 3.2 发布验收标准

满足以下条件后，可以上传体验版并提交审核：

- 数学主流程从首页到知识点详情完整可走通。
- 29 个章节都可以打开。
- 搜索、收藏、最近浏览功能可用。
- 云图片在真机上稳定显示。
- 自带内容检查脚本全部通过。
- 页面没有明显空白、死链、占位内容。
- 小程序后台服务类目和资质要求已确认。

### 3.3 发布步骤

1. 在微信开发者工具中导入项目。
2. 确认 AppID 和云环境。
3. 部署云函数 `getImageTempUrls`。
4. 上传图片资源到云存储，确认路径与 `utils/asset-config.js` 一致。
5. 编译并在模拟器完成基本浏览。
6. 使用真机预览，完整跑一遍核心路径。
7. 上传体验版。
8. 邀请内部体验成员测试。
9. 根据体验反馈修复问题。
10. 提交微信审核。
11. 审核通过后发布正式版。

### 3.4 发布后第一周观察

- 收集用户是否能顺利找到章节。
- 观察图片加载失败反馈。
- 检查搜索关键词是否命中预期内容。
- 记录高频访问章节和高频搜索词。
- 记录用户反馈的错字、错题、解析不清楚等内容问题。
- 如果审核或用户反馈集中在某类内容，优先修内容，不急着加新功能。

## 4. 第二阶段：打磨数学内容与体验

发布后第一个主要迭代方向是把数学 MVP 从“能用”打磨到“好用”。

### 4.1 内容密度统一

- 统一每章知识点数量和讲解结构。
- 统一例题层级：基础、提升、综合或压轴。
- 每章至少有明确的核心公式、易错提醒、学习路线。
- 题型模板要能回链到对应章节和知识点。
- 补充中考高频模型：动点问题、面积最值、函数交点、圆综合、相似辅助线等。

### 4.2 阅读体验优化

- 优化知识点详情页的信息层级。
- 对公式、表格、步骤、提示建立更稳定的展示规范。
- 给长题干和长解析做更好的分段。
- 图片加载失败时展示更友好的占位。
- 对章节页的学习路线做更明确的“先学什么、再练什么”。

### 4.3 搜索优化

- 支持更多别名和同义词，例如“二次函数顶点”“抛物线顶点”“顶点式”。
- 搜索结果按章节、知识点、模板分类展示。
- 增加搜索历史快捷入口。
- 增加热门关键词或推荐搜索。

## 5. 第三阶段：从知识库升级为学习工具

这一阶段的目标是让用户不只是查资料，而是形成持续学习行为。

### 5.1 错题本

- 支持把例题加入错题本。
- 记录错误原因：概念不清、计算错误、审题错误、方法不会、图形看错。
- 支持按章节、知识点、题型、难度筛选错题。
- 支持错题复习状态：待复习、已掌握、仍需巩固。

### 5.2 练习模式

- 从知识点页进入小测。
- 支持逐题作答、查看答案和解析。
- 支持基础、提升、压轴筛选。
- 支持章节综合练习。
- 支持错题自动进入错题本。

### 5.3 学习记录

- 记录已浏览知识点。
- 记录已练习题目。
- 标记掌握度：未学、学习中、已掌握、需复习。
- 生成章节学习进度。
- 我的页面展示最近学习和待复习内容。

### 5.4 复习清单

- 每章自动汇总公式、定理、易错点、典型模型。
- 支持一键进入本章错题。
- 支持考前速览模式。

## 6. 第四阶段：扩展英语和物理

数学稳定后，再扩展英语和物理。扩展前应先抽象通用内容模型，避免把所有逻辑写死在数学里。

### 6.1 通用学科模型

建议抽象为：

- 学科。
- 年级/册别。
- 章节。
- 知识点。
- 内容模块。
- 题型模板。
- 例题/练习。
- 图示/音频/表格。
- 收藏、错题、学习记录。
- 搜索索引。

### 6.2 英语方向

- 单词与短语。
- 语法知识点。
- 句型模板。
- 完形、阅读、写作题型。
- 作文模板。
- 常见易错语法。
- 音频或发音资源预留。

### 6.3 物理方向

- 概念与规律。
- 公式与单位。
- 实验题。
- 作图题。
- 计算题模板。
- 电路图、受力图、光路图。
- 与数学章节的跨学科关联，例如函数图像、比例关系、三角函数测量。

## 7. 第五阶段：内容生产与后台

长期最大的成本是内容维护，所以需要逐步从 JS 文件维护转向可管理的内容生产流程。

### 7.1 短期内容维护

- 继续使用 JS 数据文件。
- 保留校验脚本。
- 每次内容更新后运行检查脚本。
- 重要内容改动保留变更说明。

### 7.2 中期批量导入

- 支持从 Excel、CSV 或 JSON 导入章节、知识点、题目、模板。
- 自动校验必填字段、重复 ID、图片路径、题目重复。
- 自动生成搜索索引。
- 自动生成资源清单。

### 7.3 长期内容后台

- 章节管理。
- 知识点管理。
- 题目管理。
- 题型模板管理。
- 图片资源管理。
- 标签管理。
- 内容审核。
- 版本发布。
- 数据导出。

## 8. 第六阶段：账号、同步与家长视角

游客模式适合发布 MVP。等错题本、练习记录、学习进度变多后，再加入账号体系。

### 8.1 账号能力

- 微信登录。
- 收藏云同步。
- 错题本云同步。
- 学习记录云同步。
- 多设备同步。

### 8.2 家长视角

- 查看学习章节。
- 查看错题数量。
- 查看薄弱知识点。
- 查看每周学习报告。
- 推荐复习计划。

## 9. 第七阶段：智能化能力

AI 能力适合作为增强，不应在早期替代可控内容。

### 9.1 推荐方向

- 根据错题推荐知识点。
- 根据搜索词推荐相关章节。
- 根据薄弱点生成复习计划。
- 生成相似题。
- 对解题步骤做解释。
- 对英语作文或物理实验题做辅助点评。

### 9.2 风险控制

- AI 输出需要明确标识。
- 关键答案必须有可控题库兜底。
- 数学公式、图形题、证明题要避免直接给出未经校验的结论。
- 未成年人教育场景要注意内容安全和隐私保护。

## 11. 当前优先级结论

现在不要急着扩英语、物理，也不要马上做 AI。当前最重要的是：

1. 让数学 MVP 能稳定发布。
2. 让正式环境图片资源稳定显示。
3. 让核心页面没有空白、死链和占位内容。
4. 让搜索、收藏、最近浏览这些基础体验可用。
5. 发布后根据真实反馈决定下一轮迭代。

一句话路线：先发布数学 MVP，再做学习闭环，然后扩展多学科，最后建设后台、同步和智能化能力。
