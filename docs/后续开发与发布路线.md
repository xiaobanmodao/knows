# 知识通后续开发与发布路线

> 更新日期：2026-07-21
>
> 当前分支：`codex/content-packages-v1.4`
>
> 当前版本：`1.4.0-dev.1`

## 1. 产品定位

知识通定位为面向初中学生的数学、英语、物理知识查阅工具。核心价值是“快速找到、容易看懂、随时记下”，不做在线测评、打分、学完任务、单元输出任务或家长管理。

后续开发坚持四个原则：

1. 先保证内容准确、完整和可检索，再增加新能力。
2. 教材目录用于核对范围，讲解、例句、例题和插图保持原创。
3. 文字内容在图片或云服务失败时仍可完整阅读。
4. 稳定 ID 、旧链接、收藏、最近浏览和本地笔记必须向前兼容。

## 3. 立即收尾：v1.3.0

`v1.3.0` 不再扩充大量内容，只做发布收尾。

1. 提交并推送当前阅读体验分支。
2. 在 iPhone 和 Android 真机各跑一次首页、三科目录、搜索、知识页、收藏和笔记。
3. 使用弱网或断网验证文字可读、图片降级和云签名重试。
4. 更新为正式版本号，生成体验版，核对版本描述后再提审。

当前发布构建源文件约 `1049.3 KiB`，以开发者工具开启的 JS/WXML/WXSS 压缩配置试算约 `911.5 KiB`。开发者工具 2026-07-20 代码质量扫描已通过主包、代码压缩、未使用文件和敏感信息检查。

## 4. v1.4.0：分包与内容底座

这是下一个开发主线。内容将继续增长，必须先解决主包接近 `1MB` 和大型数据集中加载的问题。

### 4.1 学科分包

- 主包只保留首页、搜索索引、收藏、我的和通用组件。
- 数学、英语、物理分别进入独立分包，学科内目录、知识数据和页面同包放置。
- 搜索主包只保留轻量索引：ID、标题、别名、类型、学科和目标路由。
- 收藏、最近浏览和笔记继续使用稳定 ID，跳转时再加载相应分包。
- 验收目标：主包小于 `700 KiB`，单科分包小于 `1MB`，首页不同步加载三科全量正文。

当前开发版已经完成三科普通分包。开发者工具实际预览包体为：主包 `495.5 KiB`、英语 `268.3 KiB`、数学 `485.9 KiB`、物理 `212.8 KiB`，总计约 `1.4 MiB`；未配置自动预下载。

### 4.2 统一内容规格

- 为章节、单元、知识点、单词、语法、公式、实验和方法建立可机器校验的 schema。
- 补齐 `reviewedAt`、`sourceRefs`、`status`、`aliases` 和稳定父级关系。
- 将 `utils/math.js` 中的大型内容组装逻辑按查询、搜索索引和数据构建拆分，保留对旧 API 的兼容层。
- 内容校验输出“新增、修改、删除、引用失效”差异报告，避免目录调整时静默破坏链接。

### 4.3 兼容与回归

- 存储结构版本升级前先建立收藏、浏览位置、笔记和标签迁移用例。
- 深链接回归覆盖旧数学链接、英语单词/语法卡片定位和物理章节链接。
- 分包加载失败时给出明确重试入口，不展示空白页。

当前 8 个旧主包路径均已改为轻量跳转页，833 条主包搜索索引具备内容源哈希；v1.3 内容差异报告为新增 0、修改 0、删除 0。Nexus 5 与 iPhone 14 Pro Max 模拟器已完成三科目录和代表性内容页回归。

## 5. v1.5.0：三科内容补深

### 5.1 英语

- 336 个核心词逐词补充音标、可数性/及物性、词形变化、高频搭配、语境例句和中文辨析。
- 84 个语法点补齐结构式、使用条件、时间线或句子成分图、肯定/否定/疑问变式和易混对比。
- 单元页支持“单词、短语、语法”类型快速筛选，并保留搜索直达卡片。
- 不复制教材原文、课文或听力材料。

### 5.2 物理

- 84 个知识点逐项补齐物理量、符号、单位、公式条件、方向约定和适用边界。
- 29 个实验统一为目的、器材、操作、现象、结论、误差和安全七部分，没有的字段不用空卡片占位。
- 绘图专项复核光路、受力箭头、电流方向、电路连接点、坐标和单位。

### 5.3 数学

- 按 89 个教材小节逐项复核定义、定理条件、公式变形和图形关系。
- 对证明类、函数类、几何构造类内容增加“为什么成立”与必要条件，避免只给结论。
- 建立定理、公式、图形、模型的交叉索引，例如“勾股定理 -> 距离 -> 圆 -> 物理合力”。

### 5.4 内容验收

- 每次只合并一册或一个明确专题，不做跨三科的大批量无差别改写。
- 内容变更须同时通过结构、引用、搜索、图片和复核元数据检查。
- 抽查页面不出现通用套话、空字段占位、错误公式、语法不对应或图文无关。

## 6. v1.6.0：查阅效率与本地知识簿

- 增加数学公式索引、英语单词/语法索引、物理公式/实验索引，但不新增导航层级。
- 搜索增加命中文本高亮、别名管理、英语词形还原和常用公式符号归一化。
- 笔记支持按学科、标签和关键词筛选，支持本地导出与恢复，仍不要求登录。
- 增加“相关知识”的人工审核关系，优先补齐数学与物理的比例、图像、几何和单位换算联系。
- 增加字号、行距和图片宽度的阅读设置，仅在本地保存。

## 7. v1.7.0：内容生产工具

- 将 JS 数据的手工维护逐步迁移为结构化 JSON/CSV 导入，运行时仍生成小程序可直接加载的静态模块。
- 提供稳定 ID 生成、引用检查、搜索索引生成、资源清单和内容差异报告。
- 生成图片后自动执行尺寸、文件大小、哈希重复、引用完整性和云路径检查。
- 每条内容保留来源、复核人/方式、复核日期和变更摘要，支持待复核、已复核、已发布状态。

## 8. 暂不进入主线的能力

以下能力不进入 `v1.4-v1.7` 主线：

- 在线答题、自动评分、学完自测和单元任务。
- 错题本、掌握状态、排名、打卡和家长端。
- AI 自动生成答案、题目或未经审核的知识讲解。
- 为了同步而过早加入的强制登录。仅当本地笔记导出/恢复稳定后，才评估可选云同步。

## 9. 版本验收通用标准

1. 内容 ID 全局唯一，父子引用和旧别名有效。
2. 搜索结果能在两次操作内进入目标内容，英语单词/语法可直达卡片。
3. 云图片失败不影响正文，图示的符号、方向、单位和连线正确。
4. 旧收藏、最近浏览、阅读位置、笔记和标签迁移后仍可打开。
5. 开发者工具控制台无项目错误，iPhone 与 Android 不出现溢出、重叠或空白页。
6. 发布前通过内容、搜索、存储迁移、图片、资源清单和发布配置全量检查。

## 10. 推荐的下一步

按风险和收益排序，下一个开发任务应是：

1. 在正式云环境修复或确认图片读取权限，避免云函数签名超时。
2. 在 iPhone 与 Android 真机分别回归三科首次下载、搜索直达、收藏、笔记和弱网降级。
3. 通过后更新为 `1.4.0-rc.1`，生成体验版但不直接提交正式审核。
4. `v1.5` 先按册补深英语逐词与语法，再复核物理公式和实验，最后补数学定理关系。

一句话路线：分包底座已经完成，先过云资源与真机门槛，再按英语、物理、数学顺序持续补深。
